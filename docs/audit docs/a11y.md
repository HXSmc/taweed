### F1. [High] `tenant-switcher.tsx:51` — branch scope drops out of the accessible name below the `sm` breakpoint (WCAG 1.3.1, 4.1.2)
The current-scope indicator (`<span className="hidden text-muted sm:inline">· {scopeLabel}</span>`)
uses Tailwind `hidden` (`display:none`), which excludes it from accessible-name computation
entirely — not just visually, like `sr-only` would. **Live-confirmed** via chrome-devtools: at
1280px the trigger's real accessible name (from the page's actual a11y tree) is `"Al Salama Dental
Group · All branches"`; resized to 500px (below `sm`=640px), the same tree shows only `"Al Salama
Dental Group"` — the branch scope, which materially changes what data 4 pages show, is completely
unavailable to a screen-reader user below that width. Same defect class as the already-fixed finding
#15 (`rail.tsx`, `display:none` vs `sr-only` for nav labels).
**Fix:** GLM spoke — make the scope always present in the a11y tree (e.g. `sr-only sm:not-sr-only
sm:inline` on the existing span, or a dedicated always-present `sr-only` span alongside the visible
one), no new token/dependency needed.

### F2. [Medium] `tenant-switcher.tsx:58,66` — single-select branch list uses `menuitem`+`aria-current` instead of `menuitemradio`+`aria-checked` (WCAG 4.1.2, deviates from WAI-ARIA APG)
The branch list is a mutually-exclusive single-select rendered as plain Radix `DropdownMenuItem`s
(`role="menuitem"`) with `aria-current="true"` marking the active one, instead of the APG's
"menu offering single-select choices" pattern (`role="menuitemradio"` + `aria-checked`).
**Live-confirmed**: the rendered DOM has `role="menuitem"` + `aria-current="true"` on "All
branches" exactly as the source implies — not a total blackout (`aria-current` is commonly
announced as "current" by real screen readers), but the wrong role/state pair for "this is the
selected filter option," not "this is your current location." `components/ui/dropdown-menu.tsx`
doesn't yet export `DropdownMenuRadioGroup`/`RadioItem`, but the file already imports the full Radix
primitive (`import * as DropdownPrimitive from "@radix-ui/react-dropdown-menu"`), so adding them is
a small addition, no new dependency.
**Fix:** GLM spoke — add `DropdownMenuRadioGroup`/`DropdownMenuRadioItem` wrappers to
`dropdown-menu.tsx`, switch `tenant-switcher.tsx`'s branch list to use them.

### F3. [High] `tenant-switcher.tsx` selected-branch menu item — `text-accent` on `bg-surface-1` fails AA in dark theme (WCAG 1.4.3)
**Live-measured** via chrome-devtools (real computed styles, WCAG relative-luminance formula, not
axe): the selected item's text color `rgb(37,87,228)` (`text-accent`) on the dropdown content's
background `rgb(19,19,22)` (`bg-surface-1`) measures **3.15:1** in dark theme — below the 4.5:1
normal-text minimum. Light theme measures **5.90:1** — passes. **Same root cause as the already-fixed
finding #5** (`badge.tsx`): `--accent` is defined only in `:root` and never redefined inside `.dark`
in `app/globals.css`, unlike `--accent-subtle` which is. This is a new site of the same
already-diagnosed gap, on a component that didn't exist at the time of pass #5.
**Fix:** GLM spoke — same fix direction as finding #5 (swap to a dark-safe accent pairing for this
selected-state text, or apply the same `--accent` dark-mode token fix at the source if that doesn't
regress anything finding #5's test already locks in).

## Critical

### 1. `apps/web/components/modules/eob-review/eob-extraction-form.tsx` — line-item inputs had no accessible name at all
Every per-line claim-line input (claim line ref, SBS code, ICD-10-AM code, billed/paid/patient-
share/rejected money — 7 inputs per claim line) was a bare `<input>` with no `<label>`,
`aria-label`, `aria-labelledby`, or placeholder — WCAG 1.3.1/4.1.2 (Level A). A remittance with 3
claims × 4 lines is potentially 84 indistinguishable unlabeled fields a reviewer must correct blind
before approving money written to claims/denials.
**Fix:** visually-hidden (`sr-only`) `<span>` label added per input, mirroring the existing
`denialCode` `<select>` in the same row which already had one.
**Test:** `apps/web/test/eob-extraction-form.test.tsx`.

### 16. Chart animations ignored `prefers-reduced-motion`

**Fix:** new `apps/web/hooks/use-reduced-motion.ts` hook (mirrors the existing check in

**Test:** `apps/web/test/use-reduced-motion.test.tsx`.

### 19. `apps/web/components/money/money-figure.tsx` — trend-arrow mirroring incorrect per tone

### 23. `apps/web/components/ui/switch.tsx:13` — unchecked-state track/border near-invisible
The Switch's unchecked track (`bg-surface-3` on a `bg-surface-1` card) measured ~1.18:1 (light) /
~1.2:1 (dark) and its hairline border ~1.27:1 / ~1.3:1 — WCAG 1.4.11 Non-text Contrast requires
3:1. `Switch` had zero rendered consumers until the 2026-07-10 onboarding-corridor route wired it
in (branch toggle, step 2), so no prior audit ever reached its live DOM. In light mode the track,
border, and thumb all fail together, leaving an unchecked switch with no perceivable boundary or
state indicator at all.
**Fix:** unchecked-track fill switched from `bg-surface-3` to `bg-faint` (`--text-faint`, an
existing AA-safe-for-non-text token), clearing 3:1 in both themes without inventing a new token.
**Test:** `apps/web/test/switch-unchecked-contrast.test.tsx`.

### 24. `apps/web/components/modules/owner-report-document.tsx:73` — list loses ARIA role under Tailwind Preflight
The "Top payers recovered" `<ul>` had no `role="list"`; Tailwind Preflight's `list-style: none`
reset strips the implicit ARIA list/listitem roles in WebKit, so VoiceOver announces the rows as
unstructured text instead of a navigable list — WCAG 1.3.1. Same pattern likely exists on other
plain `<ul>`s codebase-wide (a systemic gap, not unique to this file) but this route was the first
ever scanned via a screen-reader-relevant check (axe-core can't detect WebKit's role-stripping;
the prior 25-finding pass's manual coverage was keyboard/focus-order only, not this).
**Fix:** `role="list"` added to restore the announced structure.
**Test:** `apps/web/test/owner-report-document.test.tsx`.
